# Security Policy

## 🛡️ Security First Philosophy

IBITI Guardian is built on a zero-trust model between the AI assistant and your real assets. However, because this is experimental cryptographic and artificial intelligence software, security requires shared responsibility.

---

## ⚠️ Critical Rules for Users

1. **Experimental Software**: Do not use real production funds with IBITI Guardian without thoroughly reviewing the source code and verifying its behavior in a testnet environment.
2. **Never Commit Secrets**: Never commit or upload API keys, private keys, seed phrases, keystores, or CEX secrets to any public repository or share them in unencrypted chats.
3. **Disable CEX Withdrawals**: When configuring API keys for exchanges (MEXC, OKX, Gate.io, Binance), **withdrawals must be disabled** in the exchange's API settings. IBITI Guardian only requires trading and account read permissions.
4. **Biometric Security**: Keep app passcode/biometrics active to ensure the local secure storage remains encrypted when the app is backgrounded.

---

## 🚨 Reporting Vulnerabilities

If you discover a security vulnerability, please do **NOT** open a public issue. Instead, please report vulnerabilities via GitHub Security Advisories or the official IBITI Labs contact channels.

We will evaluate and remediate reported issues as a matter of highest priority.
